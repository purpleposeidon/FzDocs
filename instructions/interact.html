Interaction Instructions


Redstone Pulse


This causes a redstone pulse to be emitted from the servo rail. Note that this can cause nearby instructions to be temporarily disabled.

Read Redstone


Measures the redstone power in the direction the servo is facing. If there is no signal, then it will push 0. If this instruction is laid on a servo rail, and it is receiving a direct redstone signal, then of course it will not work. To avoid this problem the signal must run along or underneath the servo rail.

Socket Signal


This sends a redstone signal, not to the world, but to the socket tool attatched to the servo.
It has three modes:
Socket Pulse: the socket will recieve a brief redstone signal.
Socket Powered: the socket will recieve a constant ON signal.
Socket Unpowered: the socket will recieve a constant OFF signal.
If a socket pulse is sent when Socket Powered is active, then the signal will flicker off.

Count Items


Counts how many items are held by the servo. This instruction can run without an item shifter installed.

Item Shifter Control


This instruction modifies Item Shifter settings. So of course it only works if there's an item shifter installed on the servo.
There are several modes; most of them correspond to the LMP configuration on the Item Shifter. Clicking on the instruction with an LMP will cycle through the modes.
Set Transfer Limit: reads an integer.
Set Target Slot: reads an integer; if it is -1 then the shifter will target any slot.
Probe: Counts how many items in the inventory match the target inventory. This pushes an integer.
If there was no inventory, then it pushes -1. If the item shifter had a target slot set, but that target slot was out of range of the inventory, then it pushes -1.
If the servo is holding no item, then it will count all items in the inventory.
